我做了个有点离谱的实验。
把本来只能在网页里聊天的 AI，
接进了会自己用工具干活的 Agent。

网页模型能聊，能写。
但 OpenClaw、Hermes、Claude Code、Codex 这些 Agent 工具，
要的不只是回答。
它们还要读文件，接工具，拿到结果，再继续做事。

所以我做了 WebLLM Gateway。
它把 Qwen、DeepSeek 这类网页登录模型，
整理成 Agent 客户端能调用的本地 API。
打开授权浏览器，登录网页账号，
复制地址、密钥和模型，
就可以开始接入。

Qwen 3.7 系列已经调通。
这次我让它做个真实任务：
先读项目 README，再找出卖点。

重点来了。
它不是假装读过。
模型先提出动作，
客户端按自己的权限系统真的执行工具，
把结果交回来，
模型再基于真实内容继续回答。

Gateway 做的事，说白了就是翻译。
把 Agent 想让模型做的事，翻译给网页模型。
再把网页模型的动作，翻译回 Agent 能执行的格式。
所以网页登录模型，也能进入工具调用链路。

它不是替你乱读文件，也不是绕开权限。
真正读文件、跑命令、管确认的，
始终是你的 Agent 客户端。

如果你也在折腾 Agent runtime、
网页模型接入或者 AI 编程工具，
WebLLM Gateway 已在 GitHub 开源。
