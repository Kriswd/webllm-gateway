我把 Qwen 3.7 MAX 网页模型，接进了小龙虾和 Hermes。
网页模型也能调用工具干活了？

先说清楚：
它不是拿来硬跑复杂编程长链路的。
真正适合它的，是轻中度任务。
比如整理资料、查信息、跑简单工具，
或者批量处理一些小活。

好处也很直接：
网页账号能用起来，
很多任务不用一上来就烧 API Token。

WebLLM Gateway 做的事并不玄乎。
打开授权浏览器，登录网页账号。
它自动检测可用模型，
然后给你本地 API 地址、Key 和模型 ID。

把这些填到小龙虾、Hermes 这类兼容 OpenAI 或 Anthropic API 的客户端里，
就能开始用。

重点是工具调用。
网页模型本来只会在聊天框里回文字。
Gateway 会把客户端的工具说明，
翻译成网页模型看得懂的任务。
模型需要工具时，
Gateway 再把它的动作翻译回标准工具调用。

真正执行工具的，还是你的客户端。
所以它不会绕过权限，
也不会替你乱读文件、乱跑命令。

这条链路最适合什么？
养虾养马、信息整理、简单自动化、低成本试错。
复杂工程、长时间多轮编程、强可靠交付，就别硬上。

这个项目已经开源。
核心目标就一句：
把网页登录模型变成可工具调用的 API，
让网页模型也能干点轻活，
少烧一些 Token。
