我做了个有点离谱的实验。
把本来只能在网页里聊天的 AI，
接进了会自己用工具干活的 Agent。

网页模型能聊，能写。
但 Agent 要的不只是回答。
它还要读文件，接工具，拿到结果，再继续做事。

所以我做了 WebLLM Gateway。
打开授权浏览器，登录网页账号，
复制地址和模型，
就能把 Qwen、DeepSeek 接进这些 Agent 客户端。

Qwen 3.7 已经调通。
这次我让它做个真实任务：
先读项目 README，再找出卖点。

重点来了。
它不是假装读过。
它会把动作交给客户端真的执行，
拿到真实结果以后，再继续回答。

Gateway 做的事，说白了就是翻译。
把 Agent 想让模型做的事，翻译给网页模型。
再把网页模型的动作，翻译回 Agent 能执行的格式。

真正读文件、跑命令、管权限的，
还是你的 Agent 客户端。

如果你也在折腾 OpenClaw、Hermes、
Claude Code 或 Codex，
WebLLM Gateway 已在 GitHub 开源。
