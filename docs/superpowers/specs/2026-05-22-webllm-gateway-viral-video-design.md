# WebLLM Gateway 病毒营销视频设计

日期：2026-05-22

## 目标

为 WebLLM Gateway 制作一套面向 Agent 工具玩家的开源推广视频：

- 竖屏 `9:16` 主片负责短视频扩散。
- 横屏 `16:9` 说明版负责 GitHub、B站、YouTube 和社群深度解释。
- 让不熟悉 OpenAI / Anthropic 工具协议的观众也能听懂核心价值。
- 让 OpenClaw、Hermes、Claude Code、Codex 用户一眼看出项目和自己的工具链相关。

## 受众

第一受众是正在折腾 Agent runtime、MCP、工具调用和 AI 编程客户端的用户。

第二受众是拥有 Qwen / DeepSeek 网页账号，想把网页登录模型接进现有 AI 工具链的人。

视频不把“省钱”作为第一钩子，也不承诺风控、封号或服务稳定性。视频主打的是一次可见的技术实验：本来只在网页里聊天的模型，也能接进会使用工具的 Agent 链路。

## 创意方向

选择方向：`黑客实验型`。

视觉选择：`协议破门感`，但把专业术语降到最低。画面可以保留 glitch、斜切标题、粗暴推进、真实实验日志和 UI 证据；旁白必须讲人话。

主句：

> 我做了个有点离谱的实验。  
> 把本来只能在网页里聊天的 AI，接进了会自己用工具干活的 Agent。

## 呈现方式

采用：

- AI 旁白。
- 全程大字幕，保证静音观看也能理解。
- 第一人称实验口吻。
- 黑客实验画面语言。
- 真实产品界面、模型列表、客户端接入和工具调用证据。

主叙述避免堆叠这些术语：

- `ToolBridgeV2`
- `tool_calls`
- `tool_use/tool_result`
- `provider runtime`
- `OpenAI / Anthropic compatibility`

这些术语可以在角落小字、镜头标注或横屏说明版里出现。主旁白统一翻译成：

- “把 Agent 想让模型做的事翻译给网页模型”
- “再把模型的动作翻译回 Agent 能执行的格式”
- “真正执行工具和权限确认的，还是你的 Agent 客户端”

## 时长与版本

### 竖屏主片

- 目标时长：`60-75s`
- 画幅：`1080x1920`
- 用途：小红书、视频号、抖音/B站竖版动态、微信群转发、即刻。

### 横屏说明版

- 目标时长：`90-120s`
- 画幅：`1920x1080`
- 用途：B站、YouTube、GitHub README、论坛帖和开发者社群。

横屏版可以比竖屏多讲：

- 支持 OpenClaw、Hermes、Claude Code、Codex。
- Gateway 不执行本地工具，权限仍由客户端控制。
- Qwen 3.7 模型目录和网页授权流程。

## 叙事结构

### 1. 抓停留

时长：`0-4s`

问题句：

> 网页里的 AI，能不能接进会干活的 Agent？

画面：

- 巨大斜切标题。
- 浏览器聊天框与 Agent 工具流被一条破门式转场接通。
- 第一帧必须有强动势，不能先慢慢介绍项目名。

### 2. 建立反差

时长：`4-14s`

旁白：

> 网页模型能聊、能写。  
> 但 Agent 要的不只是回答。  
> 它还要读文件、接工具、拿到结果，再继续干活。

画面：

- 左侧是网页聊天，右侧是任务流。
- 用通俗动作词代替协议图。

### 3. 揭晓项目

时长：`14-27s`

旁白：

> 所以我做了 WebLLM Gateway。  
> 登录网页账号，复制地址和模型，就能把它接进这些 Agent 客户端。

画面：

- WebLLM Gateway 首页。
- `Qwen 3.7 系列已调通` 标识。
- Qwen 授权、模型列表、接入地址和模型 ID。

### 4. 核心证据

时长：`27-47s`

旁白：

> 这次我让它做个真实任务。  
> 先读项目 README，再找出卖点。  
> 重点来了，它不是假装读过，而是真的把动作交给客户端执行。

画面：

- Agent 客户端任务。
- 工具动作被触发。
- 客户端返回工具结果。
- 最终回答引用真实 README 内容。

### 5. 亮点翻译

时长：`47-60s`

旁白：

> Gateway 做的事，说白了就是翻译。  
> 把 Agent 想让模型做的事翻译给网页模型，  
> 再把网页模型的动作翻译回来。  
> 于是网页账号，也能进入工具调用链路。

画面：

- “Agent 需求”到“网页模型”再到“Agent 动作”的双向桥接图。
- 小字标签可注明 `OpenAI`、`Anthropic`、`ToolBridge`。

### 6. 边界与转化

时长：`60-75s`

旁白：

> 真正读文件、跑命令、管权限的，还是你的 Agent 客户端。  
> 如果你也在折腾 OpenClaw、Hermes、Claude Code、Codex，  
> WebLLM Gateway 已在 GitHub 开源。

画面：

- 产品名。
- GitHub 地址。
- 项目截图。
- `Open source`、`Qwen 3.7 ready`、`Agent tool calling` 三个收束标签。

## 视觉身份

视频需要创建专用 `DESIGN.md`，基础方向如下：

- 风格：黑客实验、破门式发布片、强证据感。
- 主背景：近黑偏暖，不用普通纯黑。
- 主前景：暖白。
- 强调色：锈橙。
- 证据色：荧光绿只用于终端 / 通过标识，不成为整片主色。
- 字体：重型标题字 + 清晰字幕字。标题可压迫，字幕必须高可读。
- 动效：标题猛进、协议流破门、UI 证据快速推近、真实任务镜头停留更久。
- 避免：满屏术语、霓虹紫蓝 AI 模板感、空洞粒子流、只讲架构不展示真实任务。

## 资产

优先使用真实资产：

- Gateway 首页截图。
- Qwen 3.7 模型与授权界面截图。
- Agent 客户端真实任务截图或录屏。
- GitHub README / repo 地址镜头。

如果真实工具调用录屏在制作时还没有，先用产品 UI、任务流 UI 和真实链路字幕做预告版，但成片发布前优先补一段真实工具调用证据。

## 分发判断

### 第一梯队

- B站：横屏版主阵地，适合讲清真实技术实验并被开发者搜索到。
- GitHub README / Release / 社群帖：横屏说明版或短 GIF，直接服务 star 转化。
- 微信群、QQ群、即刻：竖屏或短横屏片段，配一句清晰钩子。

### 第二梯队

- 小红书：竖屏片，标题要通俗，主打“网页 AI 接进会干活的 Agent”。
- 视频号：竖屏片，适合微信生态内转发。
- X / YouTube：英文字幕或英文横屏版后再发，围绕 agent tool calling / OpenAI-compatible gateway。

### 选择原则

- 技术解释优先给 B站、GitHub、论坛和 Agent 社群。
- 停留与破圈优先给竖屏平台。
- 每个平台都要配仓库链接和 demo 任务，不只发视频。

## 验收标准

- 不懂协议的观众能复述一句：网页登录模型也能接进会用工具的 Agent。
- 懂 Agent 的观众能看到真实证据：授权、模型、客户端配置、工具动作、工具结果。
- 竖屏和横屏都不存在字幕裁切、文本重叠、画面空白和关键 UI 看不清。
- HyperFrames `lint`、`validate`、`inspect` 通过；新增复杂动效需要看 animation map。
- 最终产物包含可点击的本地视频路径和分发建议。
